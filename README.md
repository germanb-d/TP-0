## Descripción
En el Modelo Anemico la clase `Tiempo` solo tiene sus atributos mas getters/setters. 
Donde toda la logica se encuentra separada en otro objeto llamado `TiempoServicio`.
Esta es la manera "incorrecta" o que no cumple con los principios de POO por lo que entendi del texto.
Debido a que: "tiene todos los costos de un modelo de objetos pero ninguno de los beneficios".


En el Modelo No Anemio la clase La clase `Tiempo` contiene tanto los atributos como los métodos. 
Aunque aca se elimino directamente el Servicio, si se diera la existencia de una capa de servicios
esta no deberia implicar que un objeto quede sin comportamiento.
